# 项目启动说明

## 当前状态

✅ **后端服务器已启动** - 运行在 `http://localhost:3000`
❌ **数据库迁移未完成** - 需要先完成迁移
❌ **前端未启动** - 需要单独运行

## 解决步骤

### 1. 完成数据库迁移

在 `backend` 目录下（你已经在这个目录了），运行：

```bash
npx prisma migrate dev --name init
```

如果提示 `DATABASE_URL` 未找到，检查 `.env` 文件是否存在并包含：

```env
DATABASE_URL="file:./dev.db"
```

### 2. 启动前端（新开一个终端窗口）

```bash
# 打开新的 PowerShell 或 CMD 窗口
cd E:\project\10内容中台\frontend
npm install  # 如果还没安装依赖
npm run dev
```

前端会运行在 `http://localhost:5173`

### 3. 访问应用

- **前端页面**：http://localhost:5173 （这是主应用界面）
- **后端API**：http://localhost:3000 （API接口）
- **健康检查**：http://localhost:3000/health （测试后端是否正常）

## 常见问题

### 为什么后端地址打不开？

后端 `http://localhost:3000` 是 API 接口，不是网页界面。你需要：
1. 访问前端地址 `http://localhost:5173` 才能看到界面
2. 或者直接访问 `http://localhost:3000/health` 测试后端是否正常

### 数据库迁移失败怎么办？

确保 `.env` 文件在 `backend` 目录下，内容包含：
```env
DATABASE_URL="file:./dev.db"
```

### 前端启动失败？

确保已安装依赖：
```bash
cd frontend
npm install
```

## 完整启动流程

**终端1 - 后端**（已启动）：
```bash
cd E:\project\10内容中台\backend
# 如果还没迁移数据库
npx prisma migrate dev --name init
# 启动服务器（如果还没启动）
npm run dev
```

**终端2 - 前端**（需要新开）：
```bash
cd E:\project\10内容中台\frontend
npm install  # 首次运行需要
npm run dev
```

然后访问：**http://localhost:5173**
